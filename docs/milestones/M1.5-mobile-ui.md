# M1.5 — Mobile-friendly UI overhaul

Makes the authed application (`packages/application`) fully usable on phones and tablets. Surfaced during M1 browser verification: the desktop sidebar shell didn't collapse and data-dense pages (Movers, Watchlist, Live feed) overflowed horizontally at ~390px. Slotted between M1 and M2; the marketing site is out of scope.

## Approach / conventions

- **Breakpoint is JS-driven**, not CSS `@media`: the shared hook `app/_hooks/useIsMobile.ts` (`@/app/_hooks/useIsMobile`) wraps `matchMedia` at a 768px max-width and is SSR-safe. Components branch their inline styles on `isMobile`.
- **Inline `style={{}}` + CSS custom properties** throughout (matching the existing app); the only stylesheet touched is `app/_auth/auth.css` for the Clerk-less auth pages.
- Breakpoints exercised in QA: **phone 390×844, tablet 768×1024, desktop 1280×800.**

## Phases

### Phase 1 — Responsive app shell (#83, PR #92)

- The desktop sidebar collapses to an off-canvas **drawer** on mobile, opened by a hamburger in the top bar. The closed drawer is `inert` (not focusable/clickable). Top bar condenses.

### Phase 2 — Responsive data tables (#84, PR #93)

- Movers and Watchlist tables **reflow from rows to cards** on mobile; the dense multi-column tables are preserved on desktop/tablet.

### Phase 3 — Feed & dashboard pages (#85, PR #94)

- Live feed, the dashboard/Today page, and Analytics get responsive padding and stacking so their content fits without horizontal scroll on mobile.

### Phase 4 — Account / Alerts / forms & touch targets (#86, PR #95)

- `/account` renders Clerk's `UserProfile` **full-width** on mobile (full-width container + a mobile-only Clerk `appearance` override).
- The Watchlist card favourite **star** gains a ~40px tap target (layout-neutral `padding`/negative-margin) so it toggles reliably without opening the detail.
- The `/alerts` placeholder gets responsive padding/heading.
- (Scope note: the real **Alerts rule builder** does not exist yet — `/alerts` is a "coming soon" stub. It is built under M1 Phase 4 / #22; give it a mobile pass when it ships.)

### Phase 5 — Cross-breakpoint QA pass (#87, this phase)

- Swept every authed route (`/dashboard`, `/movers`, `/watchlist`, watchlist detail, `/live-feed`, `/analytics`, `/alerts`, `/account`) plus the public/auth pages (`/`, `/sign-in`, `/sign-up`, `/forgot-password`) at all three breakpoints. Verified the mobile drawer (focus-trapped, `inert` when closed) and tap targets.
- **Stragglers fixed:**
  - Auth pages (`/sign-in`, `/sign-up`, `/`) had a 5px horizontal overflow at 390px — the `.tb-stage` grid's implicit auto track sized to the provider buttons' max-content. Fixed structurally with `grid-template-columns: minmax(0, 1fr)`.
  - `TokenDetailPane` (the watchlist detail page **and** the slide-over overlay): the 4-col stat strip clipped the Sentiment column and the `1fr 240px` chart+dial grid pushed the dial off-screen at 390px. On mobile the stat strip is now **2×2**, the chart and dial **stack**, and the header's labelled buttons go **icon-only**.
- Verification screenshots: `docs/verification/m1.5-cross-breakpoint-p5/`.
